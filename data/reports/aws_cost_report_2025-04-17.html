Troubleshoot Amazon OpenSearch Service Charges

Follow these steps to identify and eliminate unexpected OpenSearch charges:

  1. Go to the AWS Console
  2. Sign in with your credentials
  3. Once logged in, continue to Step 2
You might need to select your account from the AWS account selector after logging in.

  1. In the AWS Console search bar, type "OpenSearch"
  2. Select "Amazon OpenSearch Service" from the results
  3. In the left navigation pane, click on "Serverless"
Make sure you're in the correct AWS region (top-right corner of the AWS Console). You may need to check multiple regions.

Common regions to check:

US East (N. Virginia) US East (Ohio) US West (Oregon) EU (Ireland) EU (Frankfurt) Asia Pacific (Tokyo)

Look for these resources that might be causing charges:

Collections

Check for active OpenSearch collections. These are the main resources that incur charges.

1. Login to AWS
2. Search for "OpenSearch" in the top bar
3. Select "Serverless" from the left menu
4. Click "Collections"
Data Access Policies

Check for data access policies that might be attached to collections generating charges.

1. Login to AWS
2. Search for "OpenSearch" in the top bar
3. Select "Serverless" from the left menu
4. Click "Data access control"
Security Configuration

Check SAML providers and encryption settings that might be causing charges.

1. Login to AWS
2. Search for "OpenSearch" in the top bar
3. Select "Serverless" from the left menu
4. Click "Security" or "Authentication"
Network Policies

Check network policies that might be maintaining connections and causing charges.

1. Login to AWS
2. Search for "OpenSearch" in the top bar
3. Select "Serverless" from the left menu
4. Click "Network" in the left sidebar

AWS might still be charging for resources you recently deleted:

  1. Go to AWS CloudTrail
  2. Click on "Event history"
  3. Add filters:
    • Event source: aoss.amazonaws.com
    • Event name: Delete
  4. Look for recent deletion events
AWS may continue billing for up to 24 hours after resource deletion.
Common Causes of Unexpected Charges
  • Unused collections that are still active
  • Resources in regions you don't typically use
  • Policies (data access, encryption, network) with no attached collections
  • Recently deleted resources that are still being billed
